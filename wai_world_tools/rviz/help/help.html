OPEN AUDITORIUM (OA) PRESENTER - Help Pages

The following pages serve as guideline how to trigger the interactions in Open Auditorium as a Presenter.

If you need help as Audience, please refer to the content at the bottom of this document.

At the moment OA supports the following input devices:

  1. [KEY] Keyboard (Key-Combinations)
  2. [PCL] Mouse (Point-And-Click and Mouse Wheel)
  3. [JOY] Joypad (Axes and Buttons)
  4. [VIV] HTC Vive (Axes and Buttons)
  5. [MOB] Mobile Devices (ROS-Mobile App)
  6. [BOI] Body Gestures (A Kinect-like 3D-Sensor is required! Pointing and activation of triggers in 3D with body gestures)
  7. [VOI] Voice Commands (Voice-To-Text Interface)
  8. [SCR] Scripted Inputs (Triggered via *.yaml script, generated from presentation slides)

Please note that all interactions with 3D data involved require a Kinect-like 3D-Sensor.




BASIC View Controls

RViz view controls

In the default view, the viewpoint is always aimed at a focal point.
You can move around the point (orbit it) or you can move the point itself.
Mouse BUTTON LEFT + Mouse MOVE Drag to rotate around the focal point.
Mouse BUTTON MIDDLE + Mouse MOVE Drag to move the focal point.
Mouse BUTTON RIGHT + Mouse MOVE Drag up or down to zoom towards or away from the focal point.
Mouse SCROLL WHEEL DOWN Zoom towards the focal point.
Mouse SCROLL WHEEL UP Zoom away from the focal point.




1. [KEY] KEYBOARD Triggers and Interactions

To trigger an interaction with the KEYBOARD, you need to use the modifier "Key TAB" or "Key CAPS LOCK".
Press and hold it and press one of the following keys.

1.1 SCENE Interactions

Key TAB + Key LEFT Select PREVIOUS SCENE.
Key TAB + Key RIGHT Select NEXT SCENE.
Key TAB + Key SHIFT Select a specific SCENE by NUMBER.

1.2 CAMERA Interactions

Key TAB + Key DOWN Transition the CAMERA (RViz) to the DEFAULT view.
Key TAB + Key UP Transition the CAMERA (RViz) to CYCLE to the next view in a list of predefined views.
Key TAB + Key Y SELECT a CAMERA (RViz) view from a list of labeled views.
Key TAB + Key Z ENFORCE the CAMERA (RViz) of the Audience/Listeners to follow the PRESENTERs camera.
Key TAB + Key ESC Toggle CAMERA (RViz) IDLE mode.
Key TAB + Key F Toggle CAMERA (RViz) mode to FOLLOW the PRESENTER. Requires a 3D-Sensor, 3D presence mode, body interaction and avatar enabled).

1.3 PRESENTER HELPER Interactions

Key TAB + Key I Toggle INTRODUCTION mode of the PRESENTER.
Key TAB + Key O Cycle through the PRESENCE modes of the PRESENTER.
Key TAB + Key B Toggle the BODY INTERACTION of the presenter (Requires a Kinect-like 3D-Sensor and the presenters PRESENCE mode set to 3D).
Key TAB + Key V Toggle AVATAR on top of the presenters head (Hope you like Tony Stark!).
Key TAB + Key L Toggle LECTERN in front of presenter (helpful if presenter is standing).
Key TAB + Key T Toggle TABLE in front of presenter (helpful if presenter is sitting).
Key TAB + Key Q Toggle AUDIO communication with Audience/Listeners.
Key TAB + Key P (Re)open the SKETCH tool. Start being creative and send your sketch to other representatives in OA.
Key TAB + Key ??? (Unmapped) Trigger 2D/3D RESPAWN, if the virtual hand pointer is close to the right half of the projectors surface.
Key TAB + Key ??? (Unmapped) Trigger applying a FORCE to a selected representative (select by moving the hand pointer close to it).
Key TAB + Key C Enable Intel and Marvin to listen to your VOICE COMMAND Interactions.
Key TAB + Key X Enable Marvin to listen to your VOICE PROMPT interactions.

1.4 AUDIENCE Interactions

Key TAB + Key INSERT REJECT (delete) next request of Audience/Listener in the queue without consideration.
Key TAB + Key HOME ACCEPT next request of Audience/Listener in the queue and focus Camera (Rviz).
Key TAB + Key S FOCUS on PREVIOUS row of Audience/Listener IDs.
Key TAB + Key W FOCUS on NEXT row of Audience/Listener IDs.
Key TAB + Key A FOCUS on PREVIOUS Audience/Listener ID.
Key TAB + Key D FOCUS on NEXT Audience/Listener ID.
Key TAB + Key < FOCUS on specific NUMBER of Audience/Listener ID.
Key TAB + Key SHIFT RIGHT SELECT/DESELECT current Audience/Listener ID.
Key TAB + Key M Send short MESSAGE to whole Audience.
Key TAB + Key N Send short MESSAGE to one Listener.
Key TAB + Key , KICK currently focused Audience/Listener ID.

1.5 EVALUATION Interactions

Note that the default fraction size for participation is a max. score of 8 and the default fraction size for examination is a max. score of 24.

The purpose of the evaluation metaphors is to increase transparency of evaluations and according accaptance on sides of the audience.

Key TAB + Key - Evaluate PARTICIPATION of currently focused Audience/Listener ID with MINUS or MINIMUM SCORE of predefined fraction.
Key TAB + Key # Evaluate PARTICIPATION of currently focused Audience/Listener ID with TILDE or MEDIUM SCORE of predefined fraction.
Key TAB + Key + Evaluate PARTICIPATION of currently focused Audience/Listener ID with PLUS or MAXIMUM SCORE of predefined fraction.
Key TAB + Key . Evaluate PARTICIPATION of currently focused Audience/Listener ID with CUSTOM SCORE of predefined fraction.
Key TAB + Key 5 Evaluate EXAMINATION of currently focused Audience/Listener ID with INSUFFICIENT of predefined fraction.
Key TAB + Key 4 Evaluate EXAMINATION of currently focused Audience/Listener ID with SUFFICIENT of predefined fraction.
Key TAB + Key 3 Evaluate EXAMINATION of currently focused Audience/Listener ID with SATISFYING of predefined fraction.
Key TAB + Key 2 Evaluate EXAMINATION of currently focused Audience/Listener ID with GOOD of predefined fraction.
Key TAB + Key 1 Evaluate EXAMINATION of currently focused Audience/Listener ID with VERY GOOD of predefined fraction.
Key TAB + Key 0 Evaluate EXAMINATION of currently focused Audience/Listener ID with CUSTOM SCORE of predefined fraction.
Key TAB + Key E Transition Camera (RViz) to see an overview of the evaluations in the EVALUATION GRAPH.
Key TAB + ??? (Unmapped) Inspect the evaluations in a specific TIME PERIOD of the currently selected Audience/Listener ID in the EVALUATION GRAPH.
Key TAB + Key 6 Prepare BOWL METAPHOR for currently focused Audience/Listener ID based on the current evaluation.
Key TAB + Key 7 Prepare MARVIN METAPHOR for currently focused Audience/Listener ID based on the current evaluation.
Key TAB + Key 8 Prepare WEIGHT BALANCE METAPHOR for currently focused Audience/Listener ID based on the current evaluation.

1.6 REP Interactions

Key TAB + Key H Focus on a specific REPRESENTATIVE and DETAIL from a list view.
Key TAB + Key G SELECT/DESELECT current representative and detail.
Key TAB + Key R Toggle ROTATION of currently selected representative and detail.

1.7 LEARNING MODE Interactions

OA is designed to implement different learning methods or modes, besides of presenting "in plenum".
Other modern and vital learning methods include, for example, cooperative learning or group-based learning.
This is still work in progress!

Key CAPS LOCK + Key 1 Switch LERANING MODE to IN-PLENUM (Frontal teaching) (default).
Key CAPS LOCK + Key 2 Switch LERANING MODE to COOPERATIVE LEARNING. The currently focused REPRESENTATIVE and according DETAILS can be investigated in a circular audience workspace layout.
Key CAPS LOCK + Key 3 Feel free to add your own cool learning method...!




2. [PAC] MOUSE (Point-And-Click) Triggers and Interactions

To trigger an interaction with the MOUSE, you need to use the key "C".
Before interacting with the RViz scene by mouse, press the C key and check if
the "Location" symbol appears. Then activate one of the following triggers.

2.1 SCENE Interactions

Note that all triggers are highlighted as transparent spherical objects, standing on a colored pier.

Key C + Click MOUSE LEFT on trigger "SCENE Previous" Transition to PREVIOUS SCENE.
Key C + Click MOUSE LEFT on trigger "SCENE Next" Transition to NEXT SCENE.

2.6 REP Interactions

Note that all representatives are highlighted with small transparent blueish spheres, located close to them.
They also represent the origin of the representatives frame.

Key C + Click MOUSE LEFT onto origin (small blueish sphere). SELECT currently focused representative and detail (DESELECT with clicking again or Key TAB+Key G).

8. [SCR] Scripted Interactions

To trigger scripted interactions, you need to place triggers inside your presentation file.
Currently, the triggers are implemented as SLIDE NOTES in each slide.

How to trigger a scripted interaction?

Scripted interactions are automatically triggered during a scene transition.
Once an according *.yaml file is available for a presentation (typically *.PPTX),
OA automatically triggers the scripted interactions for a specific scene.

SCRIPTED Interactions

The following scripted interactions are availabe inside an according session config (*.yaml) file.
If no switches are provided, OA typically uses default parameters for essential switches, which are also saved in the session config file.
Please also refer to the wai_oa_presenter_interactions_scripted session, provided in the wai_oa_gazebo/resources/sessions folder.

Projector ("projector: ") The following switches setup the PROJECTOR in OA (also refer to the examples given below!): slide ... Setup the projector to show the presentation slide for the current scene. Filename (*.png/*.PNG) ... Setup the projector to show a custom picture in the "resources/pictures" subfolder of the wai_oa_gazebo package (automatically resized to fit projector). Filename (*.mp4/*.MP4) ... Setup the projector to show a custom *.MP4 video in the "resources/videos" subfolder of the wai_oa_gazebo package (automatically resized to fit projector). livecam ... Setup the projector to show a real view from an additional external 2D camera (needs to be configured in usb_cam package first!). repcam ... Setup the projector to show a virtual view from front camera of a REP in OA (needs to be combined with "rep_name" switch!).
Examples for use: projector: slide projector: oa_demo.png projector: oa_demo.MP4 projector: livecam projector: repcam (combine it with, e.g., rep_name: marvin)
Light ("light: ") The following switches setup the LIGHT in OA (also refer to the examples given below!): off ... Setup a fully switched off environmental light. dark ... Setup an environmental light that is dark. default ... Setup an environmental light with default settings. bright ... Setup an environmental light that is bright. cold ... Setup an environmental light that is cold. warm ... Setup an environmental light that is warm. [Ambient R, G, B, Diffuse/Specular R, G, B] ... OA AUDIENCE/LISTENER - Help Pages

The following pages serve as guideline how to trigger the interactions in Open Auditorium
as Audience. Currently, the following input devices are supported:

  1. Keyboard (Key-Combinations)
  2. Joypad (Axes and Buttons)

KEYBOARD - (Audience/Listener) Triggers and Interactions

To trigger an interaction with the KEYBOARD, you need to use the modifier "Key TAB".
Press and hold it and press one of the following keys.

Basic - (Audience/Listener) Triggers and Interactions

Key TAB + Key DOWN Transition the CAMERA (RViz) to the DEFAULT view.
Key TAB + Key UP Transition the CAMERA (RViz) to CYCLE to the next view in a list of predefined views.
Key TAB + Key F Toggle CAMERA (RViz) mode to FOLLOW the PRESENTER (Your camera follows the camera of the presenter, but only if he/she updates his own camera view by trigger!).
Key TAB + Key O Cycle through the PRESENCE modes of the AUDIENCE/LISTENER.
Key TAB + Key B Toggle the BODY INTERACTION of the AUDIENCE/LISTENER (Requires a Kinect-like 3D-Sensor and the presenters PRESENCE mode set to 3D).
Key TAB + Key V Toggle AVATAR on top of the AUDIENCE/LISTENERs head (Hope you like Tony Stark!).
Key TAB + Key P (Re)open the SKETCH tool. Start being creative and send your sketch to other representatives in OA.

Requests - (Audience/Listener) Triggers and Interactions

Key TAB + Key Q Send REQUEST for a QUESTION of the AUDIENCE/LISTENER to the presenter.
Key TAB + Key R Send REQUEST for a BREAK of the AUDIENCE/LISTENER to the presenter.
Key TAB + Key M Send REQUEST for a MESSAGE of the AUDIENCE/LISTENER to the presenter.
